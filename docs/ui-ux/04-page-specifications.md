# 04 — Page specifications

**UX-PAGE-001 — Shared route contract.** Each public route has one H1, semantic hierarchy, SSR-visible localized shell, visible focus, logical layout, and status treatment from [09](09-interaction-and-page-states.md). Content routes are CURRENT route shells with PLANNED CMS/API integration; do not label a placeholder as completed content.

**UX-PAGE-002 — State applicability.** Content pages require loading, empty, recoverable error, offline, stale-data treatment, unavailable translation, and not-found where endpoint can produce it. Contact adds submitting, validation, disabled, success, unauthorized/forbidden recovery. A route with NO API REQUIRED has no loading/offline/stale state; it still handles not-found only when it is the not-found route.

**UX-PAGE-003 — Content truth.** Missing media, text, SEO, alternates, assets, or structured-data facts remain absent or use an explicit state. They never become invented claims or production placeholders.
