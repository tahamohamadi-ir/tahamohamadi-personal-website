# Patterns

**UX-DS-PATTERN-001 — Public navigation.** Navigation is persistent, labelled, locale-aware, deep-linkable, and keeps language/contact access on mobile. Route/journey ownership is [02](../02-information-architecture.md) and [03](../03-user-journeys.md).

**UX-DS-PATTERN-002 — State presentation.** Skeletons resemble final hierarchy; empty explains absence and safe next action; recovery explains cause/next step; offline stays unobtrusive; success preserves keyboard continuity. State names and priority are owned by [09](../09-interaction-and-page-states.md).

**UX-DS-PATTERN-003 — Forms.** Persistent labels, suitable input semantics, nearby announced errors, first-invalid focus, multi-field summary, one active submission, and retained recoverable input. Payload fields are contract-owned.

**UX-DS-PATTERN-004 — Editorial content.** Prose/citations/resume use editorial reading layout; repeated records use stable list/card layout. Content/media/metadata originate only from supported contracts.
